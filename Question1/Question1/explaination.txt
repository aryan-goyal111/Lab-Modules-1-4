Command: whoami
Explanation: This command displays the username of the currently logged-in user. It confirms that I am logged in using my own user account.

Command: groups
Explanation: This command lists all the groups that my user account belongs to, helping verify my access permissions on the system.

Command: pwd
Explanation: This command prints the present working directory, showing my current location in the Linux filesystem.

Command: ls -l
Explanation: This command lists all files and directories in the current directory in long format, displaying permissions, ownership, size, and timestamps.

Command: echo "Linux user environment verified" > user_info.txt
Explanation: This command creates a file named user_info.txt and writes the specified confirmation message into it.

Command: wc -m user_info.txt
Explanation: This command counts the total number of characters present in the user_info.txt file, verifying its content.

Command: man mkdir
Explanation: This command opens the manual page for the mkdir command. One useful option is -p, which allows creating parent directories if they do not already exist.

Command: ls ~ | sort
Explanation: This command lists all files and directories in the home directory and sorts them alphabetically.

Command: grep "admin" log.txt
Explanation: This command searches for the word "admin" inside the file log.txt and displays only the lines that contain the matching word.

Command: uname -r
Explanation: This command displays the Linux kernel version currently running on the system.

Command: ping -c 4 www.google.com
Explanation: This command sends four ICMP packets to google.com to verify network connectivity. Receiving replies confirms successful internet access.

Command: uptime
Explanation: This command displays how long the system has been running, the number of logged-in users, and the system load average.
